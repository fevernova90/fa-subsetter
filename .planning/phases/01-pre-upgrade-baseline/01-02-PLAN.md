From server/server.js — routes defined:
  - `GET /download` — res.sendFile(__dirname + '/generated-css/css_webfonts.zip')
    Returns 200 with file if it exists; Express sendFile sets status 404 if file not found.
  - `GET /saved-icons` — reads generated-css/saved-icons.json, res.send(data) or res.sendStatus(500)
    Returns 200+JSON if file exists, 500 if file missing or unreadable.
  - `POST /gen-webfonts` — req.body must be a non-empty array; if empty/undefined returns sendStatus(500);
    otherwise runs full pipeline (writes SCSS, runs node-sass.render, calls fontawesomeSubset, writes files, zips)
    Returns res.json(icons) on success.

server.js currently does NOT export `app`. It calls `app.listen(port)` at the bottom.
The test file will need server.js to export the `app` object WITHOUT calling app.listen.
Modify server.js to conditionally export / listen:

```js
// At the bottom of server.js, REPLACE:
//   app.listen(port, () => { ... });
// WITH:
if (require.main === module) {
  app.listen(port, () => {
    console.log('> Listening on port ', port);
  });
}

module.exports = app;
```

This is the Node.js standard pattern — server only listens when run directly, not when required by tests.
supertest will bind its own ephemeral port.

From package.json — test script:
  `"test": "react-scripts test"` — this is CRA's Jest runner for the frontend (src/).
  Server tests are plain Node.js/Jest and CANNOT run under react-scripts test (no JSX transform needed,
  different environment).
  Add a separate script: `"test:server": "jest server/server.test.js --testEnvironment node"`
  This requires jest to be available. react-scripts ships its own jest, but it is not on PATH by default.
  Use: `"test:server": "node node_modules/.bin/jest server/server.test.js --testEnvironment node"`
  supertest is not currently installed — must be installed as devDependency:
  `npm install --save-dev supertest`

jest config note: react-scripts test adds its own jest config. Running jest directly (node_modules/.bin/jest)
uses default jest config with no transforms unless configured. server.js and server.test.js are CommonJS
(require/module.exports) — no transforms needed. The `--testEnvironment node` flag is required to avoid
jsdom being injected.

    - Test "POST /gen-webfonts with empty array body returns 500": guard clause in server.js — `if (icons === undefined || icons.length === 0)` returns sendStatus(500). supertest POST /gen-webfonts with body `[]` returns 500.

  <action>
    Create `server/server.test.js` as a CommonJS module (use require, not import):

    ```js
    const request = require('supertest');
    const app = require('./server');

    describe('Server API smoke tests', () => {
      it('GET /saved-icons returns 200 or 500', async () => {
        const res = await request(app).get('/saved-icons');
        expect([200, 500]).toContain(res.status);
      });

      it('POST /gen-webfonts with empty body returns 500', async () => {
        const res = await request(app)
          .post('/gen-webfonts')
          .set('Content-Type', 'application/json')
          .send([]);
        expect(res.status).toBe(500);
      });

      it('GET /download returns 200, 404, or 500', async () => {
        const res = await request(app).get('/download');
        expect([200, 404, 500]).toContain(res.status);
      });
    });
    ```

    so the path resolves to `server/server.js`. Do NOT use `require('../server/server.js')`.

    (node-sass.render, fontawesomeSubset, file writes, archiver). Do NOT test the happy path
    in this smoke test — it requires real font files, writable filesystem, and node-sass working.
